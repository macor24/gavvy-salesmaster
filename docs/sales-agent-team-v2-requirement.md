# 销售宗师 v2.0 — 智能体销售团队需求文档

> **版本:** v2.0（草稿）
> **状态:** 需求定义阶段
> **基于:** tianlong-salesmaster v1.0.0（现有 SalesPipeline + SalesMaster 六维引擎）

## 一、产品定位升级

**从 → 到**

| 当前 v1.0 | 目标 v2.0 |
|-----------|-----------|
| 销售工具（Pipeline + 六维引擎） | 完整 AI 销售智能体团队 |
| 命令行 + HTTP API | 可视化 Web 后台 + API |
| 被动：用户手动触发 Pipeline | 主动：7×24 自动获客沟通成交 |
| 单兵作战 | 6 角色协作团队 |

## 二、智能体团队架构

### 6 个销售角色

| 角色 | 职责 | 核心能力 |
|------|------|---------|
| **1. 市场调研 Agent** | 搜索目标客户、行业分析、趋势追踪 | 关键词搜索、网页抓取、数据整理 |
| **2. 竞品分析 Agent** | 竞品监控、优劣势对比、市场定位 | 数据采集、对比分析、报告生成 |
| **3. 售前 Agent** | 需求沟通、方案定制、演示邀约 | 对话、需求分析、提案生成 |
| **4. 售后 Agent** | 客户成功、续费提醒、问题处理 | 客户跟踪、满意度调查、工单管理 |
| **5. 采购&供应链 Agent** | 供应商寻源、比价、合同管理 | 价格对比、条款审核、风险预警 |
| **6. 运营 Agent** | 销售流程优化、数据统计、KPI 看板 | 数据分析、报表生成、策略建议 |

### 协作流程

```
市场调研 Agent ──→ 发现潜在客户
       ↓
竞品分析 Agent ──→ 客户背景 + 竞品对比
       ↓
售前 Agent ──────→ 联系客户 → 需求沟通 → 提案 → 成交
       ↓
售后 Agent ──────→ 客户成功 → 续费 → 口碑
       ↓
采购&供应链 Agent → 供应商管理 → 成本优化（内部）
       ↓
运营 Agent ──────→ 全程数据监控 → 策略优化
```

## 三、功能需求

### 3.1 产品资料配置

用户通过后台输入：
- **商品名称/描述** — 卖什么
- **价格范围** — 最低价 ~ 最高价，折扣策略
- **目标客户画像** — 行业、规模、地区
- **核心竞争力** — 与竞品的差异化优势
- **常见 FAQ** — 客户常问的问题和标准回答

> 以上信息**不展示给对方（客户）**，仅销售团队内部使用。

### 3.2 自主获客

- 用户输入**关键词/行业/地区**等线索
- 市场调研 Agent 自动搜索（网页/社交媒体/招聘网站/企业信息库）
- 识别潜在客户 → 生成 Lead → 自动排入 Pipeline

### 3.3 自主沟通

- 售前 Agent 通过**各平台 API**（邮件/微信/WhatsApp/飞书等）联系客户
- 根据产品资料和客户画像生成个性化话术
- 自动跟进、提醒、记录对话历史
- 客户意向识别 → 自动升级到人工或推进到提案阶段

### 3.4 自主成交

- 根据价格范围和客户意向自动报价
- 生成合同/订单草稿
- 用户可设置"自动成交阈值"（如：报价在范围内自动确认）
- 超出阈值时通知人工介入

### 3.5 接入 DeepSeek

- 所有 Agent 的对话/分析/决策调用 DeepSeek API 增强
- 配置项：API Key、模型选择、temperature
- 降级策略：API 不可用时回退到规则引擎

### 3.6 接入各平台 API

| 平台 | 用途 | 集成方式 |
|------|------|---------|
| 邮件（SMTP/IMAP） | 发送/接收客户邮件 | smtplib / imaplib |
| 企业微信 | 客户联系 | wework API |
| 微信个人号 | 客户沟通 | 第三方桥接 |
| WhatsApp Business | 海外客户 | WhatsApp API |
| 飞书 | 客户联系 | Lark API |
| Slack | 内部通知 | Slack API |

### 3.7 Web 后台管理界面

| 页面 | 功能 |
|------|------|
| **总览看板** | 6 个 Agent 状态、Pipeline 漏斗、今日成交 |
| **客户管理** | 客户列表、沟通记录、阶段标签 |
| **商品配置** | 产品资料、价格范围、FAQ 管理 |
| **Agent 配置** | 各 Agent 开关、参数调整、DeepSeek 配置 |
| **平台集成** | API Key 管理、连接状态测试 |
| **数据报表** | 销售统计、ROI 分析、Agent 效能 |

## 四、技术架构

```
┌─────────────────────────────────────────────────┐
│                Web 后台（管理界面）               │
│  Flask/FastAPI + HTML/CSS/JS（纯标准库可选）      │
└──────────────┬──────────────────────────────────┘
               │
┌──────────────▼──────────────────────────────────┐
│             SalesOrchestrator（编排器）           │
│  负责任务调度、Agent 间通信、状态管理               │
└──────┬─────────┬─────────┬─────────┬────────────┘
       │         │         │         │
┌──────▼──┐ ┌───▼────┐ ┌──▼────┐ ┌─▼──────────┐
│市场调研  │ │竞品分析 │ │售前    │ │售后/采购/运营│
│ Agent   │ │ Agent  │ │ Agent │ │  Agent     │
└─────────┘ └────────┘ └───────┘ └────────────┘
       │         │         │         │
       └─────────┴─────────┴─────────┘
                      │
         ┌────────────▼────────────┐
         │    DeepSeek LLM 引擎     │
         │  （所有 Agent 共享调用）   │
         └─────────────────────────┘
                      │
         ┌────────────▼────────────┐
         │  各平台 API 集成层       │
         │  邮件/微信/WhatsApp/飞书 │
         └─────────────────────────┘
```

### 架构说明

- **不依赖外部框架**：优先纯标准库（参考天龙工具箱风格）
- **模块化**：每个 Agent 独立文件，可单独启用/禁用
- **降级设计**：DeepSeek 不可用时回退规则引擎
- **可扩展**：新增平台/Agent 只需添加对应模块

## 七、待确认问题

1. **Web 后台技术选型**：纯标准库 http.server（和天龙工具箱一致）还是 Flask/FastAPI？
2. **部署方式**：销售宗师独立服务？还是作为天龙工具箱的子模块？
3. **数据存储**：文件 JSON（和现有一致）还是 SQLite？
4. **多用户**：销售团队多人使用，是否需要用户/权限管理？
5. **"自主成交"的资金流**：销售宗师只出订单/合同，还是需要对接支付？
6. **首批平台**：先做邮件还是先做微信？

## 八、与现有功能的关系

- **现有 SalesPipeline**（4步流程）→ 保留，作为**售前 Agent** 的子流程
- **现有 SalesMaster**（六维引擎）→ 保留，作为各 Agent 的能力工具箱
- **现有 HTTP API** → 保留，提供给 Web 后台调用
- **LLM 引擎**（llm.py）→ 升级为 DeepSeek 集成 + 规则降级
