# 销售宗师智能体 — 完整定位与架构设计

> **版本:** v2.0
> **定位:** 四位一体 AI 销售智能体团队
> **核心理念:** 一个人就是一个销售部

---

## 一、核心定位（一句话）

> **你的AI销售团队。四位一体，自主获客、沟通、成交。**

### 四位一体释义

| 一体 | 含义 | 价值 |
|------|------|------|
| **市场洞察** | 自动搜索客户、监控竞品、分析趋势 | 不放过任何一个机会 |
| **销售执行** | 自动联系、沟通、报价、成交 | 7×24 小时在线谈单 |
| **服务交付** | 自动售后、续费、客户成功 | 让每个客户都被照顾好 |
| **运营优化** | 自动数据统计、策略调整、KPI 看板 | 持续进化越用越强 |

## 四、安全模式设计

### 为什么需要

自主成交是最大卖点，也是最大风险点。需要一套安全机制让用户放心交给 AI。

### 安全模式开关

| 模式 | 行为 | 适合场景 |
|------|------|---------|
| **🔒 保守模式（默认）** | 所有报价/成交需人工确认 | 刚上手、高客单价、谨慎型用户 |
| **🔓 开放模式** | 阈值内自动成交，超出升级人工 | 有经验、标准化产品、效率优先 |
| **🛡️ 自定义模式** | 按条件规则自动/人工 | 有明确规则的业务 |

### 安全阈值设置

```
价格天花板: ¥ [    ]    ← 超过此金额自动锁死，必须人工确认
折扣底线:   [  ] %     ← 低于此折扣自动拒绝
日成交上限: [  ] 单     ← 超过此数量自动暂停
敏感词:     [    ]     ← 客户提到这些词自动升级人工
```

### 安全日志

每一次 AI 的报价、谈判、成交动作都会记录，用户可回溯查看。
```
[2026-05-01 14:23] 售前Agent → 客户"某科技公司"
  报价: ¥12,000（阈值内 ✅ 自动确认）
  合同已生成 → 等待签署

[2026-05-01 15:01] 售前Agent → 客户"某制造企业"
  客户要求折扣 40%（超出底线 ❌ 已拒绝）
  建议: 升级到人工处理
```

## 六、产品边界

### 销售宗师 vs SentriKit

| 维度 | 销售宗师 | SentriKit |
|------|---------|-----------|
| 定位 | AI 销售智能体团队 | Agent 安全运维工具箱 |
| 市场 | 销售/业务人员 | 开发者/运维 |
| 安装 | pip install SentriKit-salesmaster | pip install SentriKit-toolkit |
| 依赖关系 | **独立运行，无需 SentriKit** | 无直接关系 |
| 可选集成 | 装 SentriKit 后可导入安全审计/竞品数据 | 不装不影响 |

**核心原则：销售宗师是独立产品，不依赖 SentriKit 就能完整运行。**
装了 SentriKit → 提案中自动包含安全审计数据，锦上添花。
没装 SentriKit → 所有功能正常，内置轻量版竞品情报和搜索能力。

## 十、竞争壁垒

| 壁垒 | 说明 |
|------|------|
| **四位一体闭环** | 市场→销售→服务→运营完整闭环，不是单一功能工具 |
| **6 角色协作** | 角色间自动传递信息，不是孤立 Agent |
| **自主成交+安全模式** | AI 自主报价谈判成交 + 可配置安全阈值 |
| **纯标准库** | 零依赖，可本地部署，数据安全 |
| **行业模板+冷启动** | 选行业即用，无需漫长配置 |
| **持续进化** | 运营 Agent 持续分析数据优化策略，越用越强 |
