# 模块合并方案：记忆库 + 能力底座

## 现状问题

- **能力底座**（abilities）：4个固定角色，硬编码静态卡片，永不变化
- **记忆库**（memory）：动态进化系统，能从实践中自动提取洞察、生成技能
- **矛盾点**：记忆库进化出的新技能永远不会出现在能力底座页面上，能力底座成了一个"死页面"

## 合并思路

能力底座应该成为记忆库中技能的**实时展示看板**，数据完全由记忆库提供。

### 数据流

```
Agent执行结果
    ↓
Learner 提取洞察（insights）
    ↓
SkillEvolver 将高置信度洞察转化为技能（skills）
    ↓
能力底座页面从 skills 集合读取并展示
    ↓
用户看见：从实践中长出来的新能力
```

### 层级关系

```
记忆库（memory/__init__.py）
 ├── MemoryKernel    — 文件数据库内核
 ├── MemoryStore     — 8集合统一存取
 ├── Learner         — 从结果中提取洞察/模式
 └── SkillEvolver    — 洞察→技能 + 技能进化 + 规则发现
         ↓
能力底座页面（abilities 视图）
 └── 从 MemoryStore 实时读取 skills 集合，动态渲染能力卡片
```

### 配置文件映射

初始4个能力从 YAML/config 中加载作为种子技能，后续进化出的新技能自动追加显示。

## 实施步骤

1. 在 `memory/__init__.py` 中导出 `get_capabilities()` 方法
2. 能力底座页面改为动态渲染（从记忆库读取 skills 集合）
3. 初始4个能力（心理学家/战略顾问/量化分析师/谈判专家）作为种子技能预置
4. 后续技能进化后自动出现在能力底座页面
