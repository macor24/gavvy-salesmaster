## 🚀 快速开始（3 步启动）

### 第一步：配置环境变量

```powershell
# 1. 复制示例配置
cd c:\Users\Administrator\.qclaw\workspace-agent-76618087\gavvy-salesmaster
copy .env.example .env
```

```powershell
# 2. 编辑 .env 文件，填入你的 API Key（最小配置）
# 至少配置 LLM API Key：
DEEPSEEK_API_KEY=sk-your-real-key
# 或
OPENAI_API_KEY=sk-your-real-key
```

### 第二步：启动数据库（可选）

```powershell
# 方式 A：使用 Docker Compose 一键启动（推荐）
docker-compose up -d postgres

# 方式 B：使用脚本启动 PostgreSQL
.\setup_postgresql.bat

# 方式 C：使用 MySQL（需要 MySQL 容器）
# 修改 .env 中 DB_TYPE=mysql，然后启动 MySQL 容器
```

### 第三步：启动服务

```powershell
# 安装 python-dotenv（如果没装）
pip install python-dotenv

# 启动服务
gavvy-sales-fastapi
```

**访问地址：** http://localhost:8877

## 🔧 环境变量配置详解

### 配置文件位置

配置文件路径：`gavvy-salesmaster/.env`

文件已创建示例：`gavvy-salesmaster/.env.example`

### 最小配置（启动必需）

```env
# ==============================================
# 最小配置（只需这一项即可使用大部分功能）
# ==============================================
DEEPSEEK_API_KEY=sk-your-real-key
```

### 完整配置选项

| 分类 | 变量名 | 说明 | 必填 |
|------|--------|------|------|
| **基础** | `DB_TYPE` | 数据库类型：`sqlite`/`mysql`/`postgres` | 否（默认 sqlite） |
| **LLM** | `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | 是（如使用） |
| **LLM** | `OPENAI_API_KEY` | OpenAI API 密钥 | 是（如使用） |
| **邮件** | `SMTP_USER` | SMTP 用户名 | 否 |
| **邮件** | `SMTP_PASS` | SMTP 密码 | 否 |
| **支付** | `STRIPE_SECRET_KEY` | Stripe 密钥 | 否 |
| **支付** | `ALIPAY_APP_ID` | 支付宝应用 ID | 否 |

## 🐳 Docker 数据库启动

### 方式 1：使用 Docker Compose（推荐）

```powershell
# 进入项目目录
cd c:\Users\Administrator\.qclaw\workspace-agent-76618087\gavvy-salesmaster

# 只启动 PostgreSQL（推荐）
docker-compose up -d postgres

# 启动全部服务（PostgreSQL + MySQL + Redis + pgAdmin）
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f postgres
```

**pgAdmin 访问（数据库管理工具）：** http://localhost:5050
- 邮箱：`admin@SentriKit.com`
- 密码：`admin123`

### 方式 2：手动启动 PostgreSQL

```powershell
# 拉取镜像
docker pull postgres:15-alpine

# 启动容器
docker run -d --name SentriKit-postgres `
  -e POSTGRES_DB=SentriKit_sales `
  -e POSTGRES_USER=salesadmin `
  -e POSTGRES_PASSWORD=sales123 `
  -p 5432:5432 `
  postgres:15-alpine

# 查看容器状态
docker ps
```

### 方式 3：手动启动 MySQL

```powershell
# 拉取镜像
docker pull mysql:8.0

# 启动容器
docker run -d --name SentriKit-mysql `
  -e MYSQL_ROOT_PASSWORD=root123 `
  -e MYSQL_DATABASE=SentriKit_sales `
  -e MYSQL_USER=salesadmin `
  -e MYSQL_PASSWORD=sales123 `
  -p 3306:3306 `
  mysql:8.0

# 然后修改 .env：
# DB_TYPE=mysql
# MYSQL_URL=mysql+pymysql://salesadmin:sales123@localhost:3306/SentriKit_sales
```

## ▶️ 启动服务

### 开发模式启动

```powershell
# 方式 A：使用命令
gavvy-sales-fastapi

# 方式 B：使用 Python
python -m SentriKit_salesmaster.app --host 0.0.0.0 --port 8877
```

### 指定配置启动

```powershell
# 临时设置环境变量（PowerShell）
$env:DB_TYPE="postgres"
$env:DEEPSEEK_API_KEY="sk-your-key"
gavvy-sales-fastapi
```

### 生产环境启动

```powershell
# 1. 配置 .env 文件
# 2. 启动 PostgreSQL
docker-compose up -d postgres

# 3. 启动服务
$env:DEBUG="false"
$env:SALES_USE_SAAS="true"
gavvy-sales-fastapi --host 0.0.0.0 --port 8877
```

## 💡 常见问题

### Q: Docker Desktop 未运行怎么办？

**A:** 使用默认的 SQLite 数据库（无需 Docker），配置：
```env
DB_TYPE=sqlite
```

### Q: 如何切换数据库？

**A:** 修改 `.env` 文件中的 `DB_TYPE`，然后重启服务：
```env
DB_TYPE=mysql  # 或 postgres，或 sqlite
```
